## 一、建校理念

### 我们为什么需要一所不存在的大学

大多数大学的官网在告诉你「我们拥有什么」——排名、实验室面积、诺贝尔奖得主人数、毕业生平均起薪。白羽大学的官网只问一个问题：**你愿意为什么而停留？**

2048 年秋天，一群散落在三块大陆上的学者、作家、系统工程师和气象学家达成了一个共识：这个世界不缺保存信息的机构。缺的是一个愿意承认「有些东西无法被数据库索引」的地方。

于是有了白羽大学。

它有三个校区。苏兹达尔的校区坐落在一座废弃的东正教钟楼里，不是因为预算有限，而是因为建校者认为，有些关于时间的学问，只有在一个能听到钟声的地方才能做好。摩尔曼斯克的校区面朝巴伦支海，极夜的漫长黑暗是它最好的观测条件——在这里工作的人，天然会思考「人为什么需要光」这种问题。屋久岛的校区在南方海岛上，川端康成研究院在此设立，理由简单到近乎任性：关于时间的某些答案，只有在一个能听到蝉鸣的地方才能找到。

白羽大学没有徽章。它用一个人——鸣濑白羽——来代表自己。一片羽毛在所有航空器结构材料中第一个被淘汰，但它至今仍然是人类最早关于飞翔这一愿望的唯一物证。白羽不提供答案，她站在那里本身就构成一个问题：在这个越来越冷的宇宙里，一个人可以给另一个人什么？

校训选的是「为世界上所有的美好而战」。建校前夕的那场争论中，有人主张用更安静的表述，但最终胜出的声音只说了两句话：「这个时代不缺保存者，缺的是愿意站出来说'这东西值得被保护'的人。」「而我们恰好有一所学校。」

### 几个你可能想知道的事实

* **春天：** 高飞纪念日当天，全校师生会把自己写过却从未寄出的信折成纸飞机，从三个校区的最高点同时掷出。不是为了庆祝什么，是因为大家发现，把一封不会抵达的信交给风是一个极具教育意义的行为。
* **夏天：** 夏末祭的篝火会烧整整七天七夜。核心规则只有一条：每个人必须带一件「本可以不说，但还是决定要说」的东西来。说完了，扔进火里。
* **秋天：** 万叶节的图书馆会在黄昏时关闭所有照明。桌子上点起蜡烛。成千上万片从三个校区收集的落叶被夹在书页之间。图书管理员说这不是浪漫主义——氧化中的叶子释放的气味，已经被证明能增强对散文的记忆保持率。
* **冬天：** 极夜节的摩尔曼斯克校区会陷入两个月的永夜。那些天里，所有课程都在烛光下进行，宿舍走廊每隔十步才亮一盏灯。这是建校以来从未改变的仪式：让每个人在彻底的黑暗中重新判断，什么是真正需要被照亮的东西。

> 白羽大学不存在。但如果你读完这些文字后感受到的东西，让你想起某个你一直想要保存却不知如何命名的夏天——那么，它在。它就在那个夏天里。

## 二、网站制作

### 技术栈

| 模块 | 技术选型与架构 |
| :--- | :--- |
| **架构** | 纯静态 SPA（单页应用），零框架，零构建工具 |
| **路由** | `hashchange` + History API，`fetch` 加载子页面内容 |
| **样式** | 原生 CSS（CSS 变量 + Flexbox / Grid + 响应式） |
| **脚本** | 原生 JavaScript（IIFE 封装，不污染全局） |
| **国际化** | 自建四语系统（中/英/日/俄），`data-i18n` 属性驱动 |
| **音频** | Web Audio API，单曲循环，跨页面不中断 |
| **部署** | GitHub Pages，纯静态托管 |

### 文件结构

```text
├── index.html          # 应用外壳（唯一入口）
├── css/
